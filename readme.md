# TeamFinder

TeamFinder — Django-приложение для поиска участников в pet-проекты. Пользователь может зарегистрироваться, оформить профиль, опубликовать проект, присоединиться к чужой идее и указать, какие навыки нужны проекту.

Главная особенность проекта — необходимые навыки проекта: автор проекта может добавлять теги навыков, а остальные пользователи могут фильтровать проекты по этим тегам.

## Возможности

- регистрация и вход по email;
- публичная страница пользователя с контактами и списком проектов;
- редактирование профиля и смена пароля;
- список пользователей с пагинацией;
- создание, редактирование и завершение проекта;
- присоединение к проекту и отказ от участия;
- блок «Необходимые навыки» на странице проекта;
- автодополнение навыков и создание нового навыка из интерфейса;
- фильтрация главной страницы по навыку проекта;
- административная панель Django.

## Стек

- Python 3.12
- Django 5.2
- PostgreSQL
- Pillow
- python-decouple

## Структура проекта

- `team_finder/` — настройки Django-проекта и корневые маршруты.
- `users/` — пользовательская модель, формы, профили, вход, регистрация и смена пароля.
- `projects/` — проекты, участие в проектах, навыки и фильтрация.
- `common/` — общие константы, валидаторы и пагинация.
- `templates/` — HTML-шаблоны приложения.
- `static/` — CSS, JavaScript, изображения и шрифты.
- `docs/manual_checklist.md` — краткий чеклист ручной проверки.

## Локальный запуск

Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv venv
venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Скопируйте пример переменных окружения:

```powershell
Copy-Item .env_example .env
```

Запустите PostgreSQL:

```powershell
docker compose up -d
```

Примените миграции:

```powershell
python manage.py migrate
```

Создайте демонстрационные данные:

```powershell
python manage.py seed
```

Запустите сервер:

```powershell
python manage.py runserver
```

Приложение будет доступно на `http://127.0.0.1:8000/projects/list/`.

## Переменные окружения

```env
DJANGO_SECRET_KEY=change_for_safety
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1

POSTGRES_DB=team_finder
POSTGRES_USER=team_finder
POSTGRES_PASSWORD=team_finder
POSTGRES_HOST=localhost
POSTGRES_PORT=5436
```

Если база уже использовалась раньше и нужно начать с чистого состояния, остановите контейнер с удалением volume:

```powershell
docker compose down -v
docker compose up -d
python manage.py migrate
python manage.py seed
```

## Тестовый аккаунт

```text
email: maria@yandex.ru
password: password
```

Команда `seed` также создаёт несколько проектов и навыков, чтобы сразу проверить фильтрацию и блок «Необходимые навыки».

## Быстрая ручная проверка

- Откройте главную страницу и проверьте список проектов.
- Нажмите один из тегов навыков и проверьте фильтрацию.
- Зарегистрируйте нового пользователя.
- Войдите под тестовым аккаунтом и откройте проект Марии.
- Нажмите «+ Добавить навык», выберите подсказку или создайте новый навык.
- Удалите навык крестиком и убедитесь, что страница не перезагружается.
- Откройте чужой проект и проверьте кнопку «Участвовать».
- Отредактируйте профиль и проверьте валидацию телефона и GitHub-ссылки.
- Создайте проект и проверьте, что автор автоматически добавлен в участники.
- Завершите свой проект и убедитесь, что статус изменился на «Закрыт».

Подробный чеклист находится в `docs/manual_checklist.md`.

## Админ-панель

Создайте суперпользователя:

```powershell
python manage.py createsuperuser
```

Админ-панель доступна по адресу:

```text
http://127.0.0.1:8000/admin/
```

В админ-панели можно управлять пользователями, проектами, участниками проектов и навыками.

## Финальная проверка

Перед сдачей рекомендуется выполнить:

```powershell
python manage.py makemigrations --check --dry-run
python manage.py check
```

Ожидаемый результат:

```text
No changes detected
System check identified no issues
```
